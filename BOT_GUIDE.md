# WB Price Monitor — Telegram Bot Edition

Полнофункциональный Telegram-бот для управления мониторингом цен Яндекс Станций на Wildberries с ноутбука.

## Что нового

✅ **Telegram-бот с управлением кнопками** — управляйте мониторингом через Telegram, не заходя в код
✅ **Long polling** — работает на ноутбуке без webhook
✅ **Фоновый мониторинг** — проверяет цены в отдельном потоке
✅ **Выбор моделей** — запускайте мониторинг конкретных моделей
✅ **Статус и отчёты** — смотрите, что происходит в реальном времени
✅ **Защита по chat_id** — бот отвечает только вам

## Команды и кнопки

### Главное меню (/start)
- 🔍 **Проверить все** — запустить разовую проверку всех 10 моделей
- ♾ **Мониторинг всех** — включить постоянный мониторинг
- 🎛 **Выбрать модели** — выбрать конкретные модели для мониторинга
- 📊 **Статус** — увидеть, запущен ли мониторинг, какие модели включены
- 🧾 **Последние цены** — список последних найденных дешёвых товаров
- ⛔ **Остановить** — выключить мониторинг (бот при этом продолжает работать)

### Выбор моделей (🎛 Выбрать модели)
- Кнопки для каждой из 10 моделей (Лайт, Лайт 2, Лайт 2 с часами, Мини, Мини с часами, Миди, Станция 2, Макс, Макс с Zigbee, Дуо Макс)
- ✅ Зелёная галочка = выбрана, ☐ пустой квадратик = не выбрана
- 🔍 **Проверить выбранные** — разовая проверка только выбранных моделей
- ♾ **Мониторить выбранные** — постоянный мониторинг только выбранных
- ⬅️ **Назад** — вернуться в главное меню

### Команды (можно писать в чат вместо кнопок)
- `/start` — главное меню
- `/status` — статус мониторинга
- `/stop` — остановить мониторинг
- `/run_all` — проверить всё прямо сейчас
- `/monitor_all` — запустить мониторинг всех моделей

## Как это работает

1. **Запускаете бот:**
   ```bash
   python main.py bot
   ```
   Бот подключается к Telegram через long polling и ждёт ваших команд.

2. **Включаете мониторинг:**
   - Нажимаете в Telegram: 🎛 → выбираете модели → ♾ Мониторить выбранные
   - Или нажимаете: ♾ Мониторинг всех

3. **Бот работает в фоне:**
   - Каждые N минут (настройка в config.yaml) проверяет выбранные модели
   - Если находит дешёвый товар — сразу отправляет уведомление в Telegram
   - Вы можете смотреть статус, запускать разовые проверки, менять модели

4. **Остановить:**
   - Нажимаете ⛔ Остановить в Telegram (или `/stop`)
   - Или нажимаете Ctrl+C в терминале

## Что происходит, если...

### Ноутбук выключился или интернет отключился
- Мониторинг останавливается
- Бот останавливается
- Данные сохраняются в `db.sqlite3`
- При перезагрузке начинаёте заново (старые цены остаются в базе)

### Ноутбук перешёл в режим сна
- Мониторинг пауза на время сна
- Возобновляется, когда ноутбук просыпается
- Нет ошибок или потери данных

### Запущены несколько копий бота
- Блокируется: нельзя запустить два мониторинга одновременно
- Сообщение в Telegram: "Мониторинг уже запущен"

## Архитектура

- **`monitor_manager.py`** — управляет фоновым мониторингом в отдельном потоке:
  - `start_all()` / `start_selected(models)` — начать проверку
  - `stop()` — остановить
  - `is_running()` — статус
  - `run_once_all()` / `run_once_selected(models)` — разовая проверка
  - `get_status()` — полный статус с результатами
  - `get_last_results()` — последние найденные товары

- **`telegram_bot.py`** — Telegram-бот с long polling:
  - Получает команды и нажатия кнопок
  - Вызывает методы `monitor_manager`
  - Отправляет статусы и результаты
  - Защита по `TELEGRAM_CHAT_ID`

- **`main.py`** — точка входа:
  - `python main.py bot` — запустить Telegram-бот
  - `python main.py run-once` — одна проверка
  - `python main.py monitor` — фоновый мониторинг без бота
  - `python main.py test-telegram` — тест Telegram
  - `python main.py show-last station_midi` — последние цены

## Настройки

В `config.yaml`:
```yaml
interval_seconds: 300        # Проверка каждые 5 минут
max_requests_per_cycle: 50   # Макс запросов за один цикл
max_requests_per_hour: 200   # Макс запросов за час
```

Интервалы для каждой модели остаются в `config.yaml`.

## Безопасность

- Бот отвечает **только** на `TELEGRAM_CHAT_ID` из `.env`
- Другие пользователи видят "Access denied" (не видят нетулл)
- Токен хранится в `.env` и игнорируется в `.gitignore`
- Long polling — данные идут через Telegram API, не через public webhook
