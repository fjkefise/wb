# Использование на ноутбуке — Шаг за шагом

## Предполагаемая ситуация

Вы хотите мониторить цены на Яндекс Станции, находясь дома или в офисе. Бот работает на вашем ноутбуке и отправляет вам Telegram-уведомления о хороших ценах.

## Установка (один раз)

### 1. Скачайте проект
```bash
git clone <repository>
cd /home/semyon/wb
```

### 2. Создайте виртуальное окружение и установите зависимости
```bash
python -m venv .venv
source .venv/bin/activate  # На Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Настройте Telegram
- Создайте бота через `@BotFather` (если ещё не создали)
- Отправьте боту любое сообщение
- Получите `chat_id` и `BOT_TOKEN`
- Создайте файл `.env`:

```bash
cp .env.example .env
# Отредактируйте с вашими значениями:
# TELEGRAM_BOT_TOKEN=123456:ABC-DEF...
# TELEGRAM_CHAT_ID=123456789
```

### 4. Настройте цены и интервалы
```bash
cp config.yaml.example config.yaml
# Отредактируйте интересующие вас модели и пороги цен
```

## Использование

### Запуск бота (каждый день)

**На Linux/Mac:**
```bash
cd /home/semyon/wb
source .venv/bin/activate
python main.py bot
```

**На Windows:**
- Двойной клик на `run_bot.bat` в папке проекта
- Или откройте PowerShell/CMD и выполните:
  ```cmd
  cd C:\path\to\wb
  .venv\Scripts\activate
  python main.py bot
  ```

Вы должны увидеть в консоли:
```
INFO - Starting Telegram bot (chat_id: 123456789)
```

Бот готов к работе!

### Управление через Telegram

Откройте чат с вашим ботом и нажимаете кнопки:

1. **🔍 Проверить все** — проверяет все 10 моделей прямо сейчас
2. **♾ Мониторинг всех** — включает проверку каждые N минут для всех моделей
3. **🎛 Выбрать модели**:
   - Выбираете модели (например, только "Лайт 2" и "Мини")
   - Нажимаете ♾ **Мониторить выбранные**
   - Теперь проверяются только выбранные модели
4. **📊 Статус** — показывает:
   - Работает ли мониторинг
   - Какие модели включены
   - Когда была последняя проверка
   - Сколько найдено товаров и отправлено уведомлений
5. **🧾 Последние цены** — список последних дешёвых товаров
6. **⛔ Остановить** — выключает мониторинг (но бот продолжает работать)

### Примеры сценариев

**Сценарий 1: Простой мониторинг всего**
1. Запустили бот
2. Нажали ♾ **Мониторинг всех**
3. Бот проверяет все модели каждые 5 минут (или N минут по config)
4. Когда найдётся хорошая цена — Telegram отправит уведомление
5. Когда уходите — нажимаете ⛔ **Остановить** или Ctrl+C в консоли

**Сценарий 2: Мониторим только интересующие нас модели**
1. Запустили бот
2. Нажали 🎛 **Выбрать модели**
3. Выбрали "Лайт 2 с часами" и "Макс"
4. Нажали ♾ **Мониторить выбранные**
5. Бот проверяет только эти две модели

**Сценарий 3: Разовая проверка перед отключением**
1. Нажали 🔍 **Проверить все**
2. Бот проверил все модели один раз
3. Я увидел результаты в Telegram
4. Нажал ⛔ **Остановить**
5. Закрыл бот (Ctrl+C)

## Что происходит в фоне

Когда вы включили мониторинг (♾ **Мониторинг всех** или **Мониторить выбранные**):

1. Бот запускает фоновый поток
2. Каждые N минут (из config.yaml) поток проверяет выбранные модели
3. Для каждой модели:
   - Ищет товары на Wildberries
   - Сравнивает цены с историей
   - Если цена хорошая или упала сильно — отправляет Telegram-уведомление
4. Между запросами — задержки для того, чтобы не перегрузить WB
5. Все цены сохраняются в базе `db.sqlite3`

Вы можете в любой момент:
- Посмотреть статус: 📊 **Статус**
- Запустить разовую проверку: 🔍 **Проверить все**
- Изменить выбранные модели: 🎛 **Выбрать модели**
- Остановить: ⛔ **Остановить**

## Когда компьютер выключен / интернет отключен

**Что произойдёт:**
- Мониторинг остановится
- Бот завершит работу
- Вы **не будете получать** уведомления

**Когда вы снова включите компьютер и запустите бот:**
- База данных с историей цен сохраняется в `db.sqlite3`
- Бот начнёт с чистого листа, но статистика останется
- Старые цены будут учитываться при расчёте медианы

**Если вы хотите, чтобы бот работал постоянно:**
- Используйте VPS или облако (см. раздел про systemd/Docker в README_BOT.md)
- Или оставляйте ноутбук включённым 24/7

## Как держать бот работающим при закрытии терминала (Linux/Mac)

### Способ 1: screen (простой)
```bash
screen -S wb_monitor
# Запускаете бот:
python main.py bot
# Закрываете окно: Ctrl+A D (отделяете сессию)
# Позже вернётесь:
screen -r wb_monitor
# Остановка:
screen -X -S wb_monitor quit
```

### Способ 2: tmux (более удобный)
```bash
tmux new-session -s wb_monitor
# Запускаете:
python main.py bot
# Закрываете: Ctrl+B D
# Вернётесь:
tmux attach-session -t wb_monitor
```

### Способ 3: systemd (для постоянной работы)
Смотрите `examples/wb-monitor.service` и README_BOT.md

## Windows: Запуск через Task Scheduler

Если хотите, чтобы бот автоматически запускался при включении ноутбука:

1. Откройте **Task Scheduler** (Планировщик задач)
2. Create Basic Task (Создать базовую задачу)
3. Trigger: When computer starts (При запуске)
4. Action: Start a program (Запустить программу)
5. Program: `C:\path\to\project\run_bot.bat`
6. Enable: Run whether user is logged on or not (Работать в фоне)

## Логи и отладка

Логи пишутся в:
- **Консоль** — вы видите в реальном времени
- **`wb_monitor.log`** — файл с историей

Если что-то не работает:
1. Посмотрите в консоль — там будут ошибки
2. Проверьте `.env` — правильный ли токен и chat_id
3. Запустите тест: `python main.py test-telegram`
4. Посмотрите в `wb_monitor.log`

## Смена порогов цен "на лету"

Хотите изменить интересующую вас цену или интервал проверки:

1. Остановите бот: ⛔ **Остановить** или Ctrl+C
2. Отредактируйте `config.yaml`
3. Запустите бот снова: `python main.py bot`

Новые настройки вступят в силу при следующей проверке.

## Надёжность

- **Если WB вернул ошибку 429 или 5xx:** Бот сделает паузу и повторит позже (exponential backoff)
- **Если потеряется интернет:** Бот будет ждать восстановления, не вылетая
- **Если Telegram недоступен:** Уведомления будут переотправлены при восстановлении
- **Если одна модель сломалась:** Остальные продолжат работать
